# 4. 1 Overfitting 
---
Overfitting refers to a situation where a model performs well on the training data but fails to generalize to unseen data because it has focused too much on the training set. Visually, an overfitted model would appear too wiggly or distorted to pass through every data point instead of fitting the general trend.

## 4. 1. 1 Training Set / Validation Set / Test set
---
Our first step is to divide the dataset into three splits. 60~80% of the data is used as the **training set**, which is used to train the model. During this process, you'll likely want to adjust the hyperparameters multiple times to improve the accuracy. At this point, you adjust the hyperparameters in a way that minimizes the cost on the **validation set**. **Test set** is only used for final evaluation not modifying the model.

![figure4-1][ML4-1.png]

The test set is used as an indicator of a model’s ability to generalize. Therefore, selecting a model based on which one performs best on the test set may result in overfitting to that test set.

## 4. 1. 2 Regularization
---
As the absolute values of the weights for multiple features, especially polynomial features, grow larger, the model becomes more curved and complex. Therefore, during the calculation of the cost, we can add a penalty term to the weights to restrict their values from becoming too large.

$$\begin{aligned}
J(\vec{w}, b) = L(\hat{y}, y) + \underbrace{\frac{\lambda}{m} \sum_{j=1}^{n} w_j^2}_{\text{Reg. term for } w_j} + \underbrace{\frac{\lambda}{m} b^2}_{\text{Optional Reg. term for } b}
\end{aligned}$$

,where $\lambda$ is a [[4. Diagnostics#Hyperparameter|hyperparameter]] that determines how much penalty to give to the weights.
## 4. 1. 3 Dropout
---
Dropout is one of the solutions to overfitting in neural networks, especially deep learning, which helps to prevent the model from relying too heavily on specific neurons. When dropout is applied to a fully connected layer, each neuron is excluded from training with a probability equal to the dropout rate.

# 4. 2 High Bias / High Variance
---
High bias means underfitting, and high variance means overfitting. Underfitting means the model is too simple, resulting in high cost not only on unseen data but also on the training set.

When a model encounters such problems, one possible approach is to simply try to collect more data to train. Let's take a look at how effective that might be in each problem.

## 4. 2. 1 Polynomial Feature Plot
---
Let $d$ be the highest degree of the features. Then, the cost corresponding to $d$ is as follow:

![figure4-2][ML4-2.png]
1. High bias(underfit)
	- $J_{train}$ is high
	- $J_{val}$ is also high
	- Increase $d$
	- Similarly, you can get additional features.
2. High variance(overfit)
	- $J_{val}>>J_{train}$
	- Decrease $d$
	- Reduce some unnecessary features.
	- **If you don’t want to lose certain features, you might consider using regularization.**

We need to adjust the degree $d$ to find the tradeoff between high bias and high variance.

## 4. 2. 2 Regularization Parameter Plot
---
Adjusting the regularization hyperparameter can also cause high bias or high variance problem as follow:

![figure4-3][ML4-3.png]

	- $J_{train}$ is high
	- $J_{val}$ is also high
	- Decrease $\lambda$
We need to adjust $\lambda$ to find the tradeoff between high bias and high variance.

# 4. 4 Skewed Dataset Problem
---
Now, let’s look at the problems that can occur with skewed datasets. Suppose some rare disease diagnosis model has 99% accuracy, 1% error on test set and the percentage of actual patients is 0.5%. Then there is much more simple algorithm that has higher accuracy(0.5% error).

```python
def diagnosis():
    print("y = 0(negative for test)")
```

Model just saying all subjects that tested negative has higher accuracy. Let's correspond this skewed data problem.

## 4. 4. 1 Error Matrix
---

| Predicted \ Actual |       1(Pos.)        |       0(Neg.)       |
| :-----------------: | :------------------: | :-----------------: |
|       1(Pos.)       | **True Pos.**<br>15  | **False Pos.**<br>5 |
|       0(Neg.)       | **False Neg.**<br>10 | **True Neg.**<br>70 |
Above table shows more than the accuracy that reflects how well the predictions match the targets.

In Above diagnosis model,
- True Positive : Diagnose actual patients as positive -> 15
- False Positive : Diagnose actual patients as negative -> 5 (misclassified)
- False Negative : Diagnose non-patients as positive -> 10 (misclassified)
- True Negative : Diagnose non-patients as negative -> 70

But above simple model has only 25 false Neg, 75 true Neg. To distinguish between two models more effectively, the metrics **precision** and **recall** were devised.

## 4. 4. 2 Precision & Recall
---
$$\begin{aligned}
\text{precision} & =\cfrac{\text{True Pos.}}{\text{Predicted Pos.}}=\cfrac{\text{True Pos.}}{\text{True pos.}+\text{False Pos.}} = 0.75\\
\text{recall} & =\cfrac{\text{True Pos.}}{\text{Actual Pos.}}=\cfrac{\text{True Pos.}}{\text{True pos.}+\text{False Neg.}} = 0.6
\end{aligned}$$

- **Precision** measures how many of the predicted positives are actually positive.
- **Recall** measures how many of the actual positives were predicted to be positive.

Unlike the model shown in the table, the "y=0" model has 0 precision because it never predicts any positive cases.

## 4. 4. 3 Trading off Precision and Recall
---
Since both precision and recall are significant, we should get higher precision and recall values. Suppose following two situations:

1. Precision is much considerable than recall
	- In rare disease diagnosis, if patients who test positive have to pay expensive additional tests or treatments with side effects, it is important to minimize **false positive** diagnoses as much as possible.
	- Then, we can increase the classification threshold.

2. Recall is much considerable than precision
	- On the other hand, suppose that if the disease is detected early, it can be treated easily and cheaply with minimal side effects, but if missed early, it can become very severe.
	- So we want to minimize **true negative** diagnoses as much as possible.
	- Then, to reduce misclassification, we can decrease the classification threshold.

You can manually adjust the threshold case by case, or use **F1 Score**

## 4. 4. 4 F1 Score
---
F1 score is a metric used to compare the precision and recall of multiple models at once. F1 score is actually the harmonic mean of precision and recall, not the arithmetic mean. The reason for using harmonic mean is as follow:

|         | Precision | Recall | Arithmetic Mean | F1 Score  |
| :-----: | :-------: | :----: | :-------------: | :-------: |
| Model 1 |    0.5    |  0.4   |      0.45       | ==0.444== |
| Model 2 |    0.7    |  0.1   |       0.4       |   0.175   |
| Model 3 |   0.02    |  1.0   |    ==0.501==    |  0.0392   |
1. Arithmetic Mean
	- Model 3 has highest average of precision and recall but has low precision
	- That is, arithmetic mean cannot distinguish a model where one side is very low.
	- Actually, Model 3 is nearly "y=0" model.
2. F1 Score(Harmonic Mean)
	- $\text{F1 score}=\frac{2\text{Precision}\cdot\text{Recall}}{\text{Precision}+\text{Recall}}$
	- It is seem to be rational to choose Model 3 with F1 Score.
	- F1 score emphasis the lower value between precision and recall.

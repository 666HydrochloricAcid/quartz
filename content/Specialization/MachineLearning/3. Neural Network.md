# 3. 0 Introduction
---
Neural networks are machine learning models that inspired from the architecture of the biological brain. The biological brain operates through the following steps:
1. External stimuli are received, and information (signals) is transmitted to neurons.
2. If the received signal exceeds a certain threshold, it is sent to other neurons through the synapse.

Suppose that we are building a demand prediction model for electronic products with features such as price, specifications, design, brand name, and shipping cost. We will compute the economicality, rationality, and suitability of this data in our own way. Each of the calculated ones will again make an instinctive calculation to decide whether to purchase as below.

![[ML3-1.png]]
- The layer on the left is the **Input layer**, and in the example above, each unit receives inputs labeled price, specifications, design, brand name, and shipping cost.
- The middle layer is the **Hidden layer**, and each unit represents e, f, and g, respectively.(In fact, we don't actually know what each of these represents.)
- The final layer is the **Output layer**, where 1 indicates a purchase and 0 indicates no purchase.

The signal arriving at each node is a linear combination of the signals from which the arriving arrow originates. Therefore, if we look only at the hidden layer to output layer connection, we can see that it is a 3-featured logistic regression.

We can build models even with more hidden layers, which allows them to recognize more complex patterns. And we can even add more **neurons (or nodes, units)** per each layer, which are represented by circles in the diagram, to expand the model's representational capacity.

# 3. 1 Model Architecture
---
First, we would clarify the numbering of layers as follow:
- Input Layer - Layer 0
- 1st Hidden Layer - Layer 1
- nth Hidden Layer - Layer n
- Output Layer - Layer n+1

Each neuron in each layer has weight and bias denoted by $w$ and $b$, respectively. For example, the $j$ th neuron in the $l$ th layer has parameters $\vec{w}^{[l]}_j$​ and $b^{[l]}_j$ as an weight and a bias, respectively.

Then, for given special function $g$, $k$ neurons in the 1st layer outputs the following values:

$$
\begin{aligned}
a^{[1]}_1=g & \left(\vec{w}^{[1]}_1\cdot\vec{x}+b^{[1]}_1\right) \\
a^{[1]}_2=g & \left(\vec{w}^{[1]}_2\cdot\vec{x}+b^{[1]}_2\right) \\
&\vdots \\
a^{[1]}_k=g & \left(\vec{w}^{[1]}_k\cdot\vec{x}+b^{[1]}_k\right)
\end{aligned}
$$
These values are called **activations** because they represent the signal that each neuron sends to the next neuron. Or, we can vectorize these values to calculate the first layer’s activations all at once, as follows:

$$
\vec{a}^{[1]}=\left(a^{[1]}_1,a^{[1]}_2,\cdots,a^{[1]}_k\right)=g\left(\vec{w}^{[1]}\cdot\vec{x}+b^{[1]}\right)
$$

If we define the input vector as $\vec{a}^{[0]}$, the activation of the $j$ th neuron in the $l$ th layer can be generalized as follow:

$$
\vec{a}^{[l]}_j=g\left(\vec{w}^{[l]}_j\cdot\vec{a}^{[l-1]}+b^{[1]}_j\right)
$$

# 3. 2 Forward Propagation
---
The process of moving through the neural network in the forward (rightward) direction to obtain the output $f(x)\left(=\vec{a}^{[L]}\right)$ is called forward propagation (,where $L$ is the last layer). As before, this can be interpreted as the model’s prediction.

## 3. 3. 1 ReLU Function
---
Rectified Linear Unit or ReLU is one of the most commonly used activation functions in the hidden layers of NN. ReLU $g:\mathbb{R}\rightarrow[0,\ \infty)$ is defined as follow:

$$
g(z)=\max(0,z)
$$
If a neuron in the hidden layer is intended to represent awareness of a product, there would be no activation function more suitable than ReLU. This is because awareness can increase linearly, but negative awareness is not a valid concept. Additionally, ReLU has the following characteristics:
- Fast Calculation : 0-zone makes calculation more faster
- Only 1 flat zone (in graph) : Otherwise, sigmoid has two flat zones and flat zone slows down the speed of gradient descent

## 3. 3. 2 tanh Function
---
While tanh and sigmoid share a similar S-shaped curve, tanh is zero-centered, unlike sigmoid which is centered at 0.5. This helps address some of the issues that arise when using sigmoid in gradient descent, though we will not go into detail here. Let us simply review the definition of tanh and move on.

$$
\tanh{z}=\cfrac{e^z - e^{-z}}{e^z + e^{-z}}
$$
## 3. 3. 3 Softmax Function & Multi-Class Classification
---
Softmax is a representative activation function used in the output layer. It extends binary classification to multi-class classification. If we are building a model to classify handwritten digits, we would place 10 neurons in the output layer—each representing a prediction for digits 0 through 9. By applying the softmax function as the activation, the outputs can be interpreted as the predicted probabilities for each digit.

$$
\text{softmax}(z_j) = a_j = \cfrac{e^{z_j}}{\sum_{i=1}^N e^{z_i}}
$$


where the model is classifying into $N$ classes, and $a_j$ represents the probability of being classified into the $j$ th class(denoted as $P(y=j)$ in statistics).

Additionally, the loss can be derived as an extension of CEE in binary classification, and is defined as follow:

$$
\begin{aligned}
L(a_1, a_2, \cdots, a_N) =
\begin{cases}
-\log a_1 & \text{if, } y=1 \\
-\log a_2 & \text{if, } y=2 \\
\qquad\qquad\vdots \\
-\log a_N & \text{if, } y=N
\end{cases}
\end{aligned}
$$


This allows us to build a multi-class classification model.

# 4. Optimizer
---
When the amount of data is small and the neural network is shallow, gradient descent is often sufficient to achieve good performance.   However, as the data size increases and the network becomes deeper—as in deep learning—the choice of optimizer becomes significantly more important. Let's take a look at how other optimizers, while following the basic structure of [[1. Linear Regression Model#1. 2 Gradient Descent|gradient descent]], have evolved.

## 4. 1 Stochastic Gradient Descent(SGD)
---
Recently, as the amount of data has increased, SGD is often treated as the default optimizer. The following outlines how SGD works:

1. Shuffle the training data (optional but common).
2. Randomly sample a small subset of the training data.
3. Compute the loss and the gradient on the sampled data.
4. Update the parameters using a given learning rate.
5. Repeat until convergence or until stopping criterion is triggered.

SGD can perform faster computations by using a sampled subset of data at each update, rather than the entire dataset. As a result, some updates may be in inaccurate directions, but this also increases the chance of escaping local minima. Furthermore, as will be discussed later, this uncertainty can help address the problem of overfitting.

## 4. 2 Momentum
---
Momentum is an extension of SGD by introducing the concept of inertia. We define a velocity based on previous descent steps and applies that velocity into the current update.

$$
\begin{aligned}
v_t & = \gamma v_{t-1}+\alpha\cfrac{\partial J}{\partial(w,b)}\\
(w,b) & =(w,b)-v_t
\end{aligned}
$$
where $\gamma$ is the friction coefficient, which controls how much of the previous velocity is used in the current ($t$ th) update.

The inertia helps prevent slow convergences caused by oscillations, and makes it easier to escape from local minima or saddle points.

## 4. 2 Adaptive Gradient(AdaGrad)
---
While Momentum optimizes directionality, AdaGrad focuses on speed, adjusting learning rates based on previous steps. The idea is to start with a large learning rate for fast movement toward the global minimum, then reduce it gradually for finer adjustments.

$$
\begin{aligned}
h_t & =h_{t-1}+\cfrac{\partial J}{\partial(w,b)}\odot\cfrac{\partial J}{\partial(w,b)}\\
(w,b) & =(w,b)-\alpha\cfrac{1}{\sqrt{h_t}}\cfrac{\partial J}{\partial(w,b)}
\end{aligned}
$$

$h$ is a measure that reflects how much each parameter was updated in the previous steps. By multiplying by its inverse, the learning rate can be adjusted for each parameter accordingly(or it is also referred to as learning rate decay. ). By reducing the learning rate for parameters that have been updated considerably, this technique makes convergence more faster.

In fact, since AdaGrad's learning rate converges to zero with repeated updates, RMSProp was introduced to overcome this limitation. However, this will not be discussed in detail here.

## 4. 3 Adaptive Moment Estimation(Adam)
---
Adam is an algorithm that combines the ideas of Momentum and RMSProp. Specifically, it applies momentum to the moving direction while also adapting the learning rate based on recent changes in the curvature of the loss surface.

It has been shown to perform well across a wide range of neural network architectures and is currently one of the most commonly used optimization algorithms. However, this will not be discussed in detail here.

# 5. Backward Propagation
---
Backward propagation(or Backprop) is the process of determining how much each parameter in the model contributes to the cost $J$. In other words, backprop is is one of the methods to find values of $\frac{\partial J}{\partial w}$ and $\frac{\partial J}{\partial b}$ . The figure below illustrates the processes of forward propagation and backward propagation.

![[ML3-3.png]]

As illustrated in the figure, backprop computes partial derivatives in the reverse order of the forwardprop by systematically applying the chain rule Mathematically, this can be expressed as follow:

1. 

$$
\cfrac{\partial}{\partial d}J = d
$$

2. 

$$
\cfrac{\partial}{\partial a}J = \cfrac{\partial J}{\partial d}\cfrac{\partial d}{\partial a}=d\cdot1
$$

3. 

$$
\begin{aligned}
\cfrac{\partial}{\partial b}J & = \cfrac{\partial J}{\partial d}\cfrac{\partial d}{\partial a}\cfrac{\partial a}{\partial b}=d\cdot1\cdot1 &(1)\\
\cfrac{\partial}{\partial c}J & = \cfrac{\partial J}{\partial d}\cfrac{\partial d}{\partial a}\cfrac{\partial a}{\partial c}=d\cdot1\cdot1
\end{aligned}
$$

4. 

$$
\cfrac{\partial}{\partial w}J = \cfrac{\partial J}{\partial d}\cfrac{\partial d}{\partial a}\cfrac{\partial a}{\partial c}\cfrac{\partial c}{\partial w}=d\cdot1\cdot1\cdot x \qquad (2)
$$

# 6. Convolutional Layer
---
So far, we have dealt with layers in which each neuron is connected to all activations from the previous layer. This is called a fully connected layer, and since it flattens the data into a one-dimensional vector, it is relatively unsuitable in cases where the structure or patterns within the data are important(image, time series data, ...). In a convolutional layer, each neuron is connected only to a local region of the input, determined by the size of the filter. The filter slides over the input according to a defined stride, and at each position, the neuron processes the activations. More detailed explanations will be covered separately later.

# 1. 0 Introduction to Regression
---
A linear regression model (or linear model for regression) is a model that predicts an output for a sample represented by a **continuous number** or a vector of continuous numbers.
Let's dive into the Housing Problem

### § Housing Problem
![feature 1](ML1_1.png)

Problem Statement : There are five data points, each containing the square footage and the price of a house. Given the square footage of a new house, can we predict its price?
### Data Table

| :-: | :-: |
|  1  | 300 |

### Notation
- $x$ : **feature**, input variable (size)
- $y$ : **target**, output variable (price)
- $m$ : number of training examples ($m=5$)
- $(x^{(i)}, y^{(i)})$ : $i$ th training example ($(x^{(1)}, y^{(1)}) = (1, 300)$)

### Model
In this problem, the model takes the size as input and outputs the predicted price, say $\hat{y}$.
We represent model $f$ as follow:
$$f_{w,b}(x)=wx+b(=\hat{y})$$

# 1. 1 Cost Function
---
Eventually, our goal is to find the best parameters $w$ and $b$ such that the prediction of $i$ th sample $\hat{y}^{(i)}$ is close to the target value $y^{(i)}$ for all samples. Hence, we quantify the errors between predictions and targets, which is referred to as the cost function defined as follow:
$$
\begin{aligned}
J(w,b) &= \cfrac{1}{m} \sum_{i=1}^{m} \left( \hat{y}^{(i)} - y^{(i)} \right)^2 \\
       &= \cfrac{1}{m} \sum_{i=1}^{m} \left( f_{w,b}(x^{(i)}) - y^{(i)} \right)^2
\end{aligned}
$$

where $\left(f_{w,b}\left(x^{(i)}\right)-y^{(i)}\right)$ is called error term. This cost function is called the **Mean Squared Error** function or **MSE Loss**.
Our goal is to find $w$ and $b$ that minimize the value of $J$ : $min_{w,b}J(w,b)$
# 1. 2 Gradient Descent
---
#Optimization
If we plot $J(w,b)$ with respect to $w$ and $b$, we obtain the graph as below.
_I plotted a red point with random values of $w$ and $b$ as our initial parameters._
![feature 3](ML1_3.png)
Shifting the red point to a lower position on the surface corresponds to minimizing the value of cost function. With partial derivatives of $J(w,b$) with respect to $w$ and $b$, let's determine the direction and magnitude of the step the red point should take.

1. Set an appropriately small number called **learning rate**, $\alpha=0.001$.

2. Update parameters as follow:
$$
\begin{aligned}
w - \alpha \cfrac{\partial}{\partial w} J(w,b) \quad \overset{\text{sub}}{\rightsquigarrow} \quad w \\
b - \alpha \cfrac{\partial}{\partial b} J(w,b) \quad \overset{\text{sub}}{\rightsquigarrow} \quad b
\end{aligned}
$$

3. Repeat step 2 until the red point arrives near the minimum.
4. We now know how to set $w$ and $b$.
### Learning Rate
Improper value of learning rate can cause...
- slow updates when $\alpha$ is too small
- Slow updates may require a higher number of iterations to converge to minimum, which can increase the computational cost.
- overshoot the minimum when $\alpha$ is too large
- Overshoots can make the point not to converge but either diverge or oscillate.

# 1. 3 Multiple Linear Regression
---
The motivation for the problem is that the price of a house depends not only on its size but also on other factors. That is, we can use other features(floors, ages, ...) to carry out more accurate predictions.

Previous Model : $f_{w,b}(x)=wx+b$
With multiple features : $f_{\vec{w},b}(\vec{x})=\vec{w}\cdot\vec{x}+b=w_1x_1+w_2x_2+\cdots+w_nx_n+b$, where each sample has n features

### Notation
- $x_j$ : $j$ th feature
- $n$ : number of features
- $\vec{x}^{(i)}$ : feature vector of $i$ th training sample
- $x^{(i)}_j$ : value of $j$ th feature in $i$ th training sample

### § Expanded Housing Problem with data table
| $x_1$(price) | $x_2$(bedrooms) | $x_3$(floors) | $x_4$(age) | $y$ |
| :----------- | :-------------- | :------------ | :--------- | :-- |
| 1            | 2               | 2             | 7          | 300 |
| 1.2          | 3               | 1             | 10         | 375 |
| 1.6          | 3               | 1             | 20         | 450 |
| 1.8          | 2               | 2             | 15         | 475 |
| 2.0          | 3               | 2             | 15         | 500 |
### Vectorization
Let's reformulate the model using vector notation.
- parameters : $\vec{w}=(w_1, w_2, \ldots, w_n),\ b(scalar)$
- model : $f_{\vec{w},b}(\vec{x})=\vec{w}\cdot\vec{x}+b$
- cost function : $J(\vec{w},b) = \frac{1}{m}\sum^{m}_{i=1}\left(f_{\vec{w},b}(\vec{x}^{(i)})-y^{(i)}\right)^2$
- gradient descent : $\begin{matrix}\vec{w} - \alpha \frac{\partial}{\partial \vec{w}} J(\vec{w},b) \quad \overset{\text{sub}}{\rightsquigarrow} \quad \vec{w}\\b - \alpha \frac{\partial}{\partial b} J(\vec{w},b) \quad \overset{\text{sub}}{\rightsquigarrow} \quad b\end{matrix}$
### Gradient Descent in Multiple Linear Regression

The partial derivative with respect to a vector $\vec{w}$ is taken element-wise, resulting in a vector of partial derivatives. That is, the operations are as follow:

$$\cfrac{\partial}{\partial \vec{w}} J(\vec{w},b)=\left(\cfrac{\partial}{w_1} J(\vec{w},b),\ \cfrac{\partial}{w_2} J(\vec{w},b),\ldots,\ \cfrac{\partial}{w_n} J(\vec{w},b)\right)$$
Therefore, gradient descent is performed as follow:

$$\begin{matrix}w_1 - \alpha \cfrac{\partial}{w_1} J(\vec{w},b) \quad \overset{\text{sub}}{\rightsquigarrow} \quad w_1\\w_2 - \alpha \cfrac{\partial}{w_2} J(\vec{w},b) \quad \overset{\text{sub}}{\rightsquigarrow} \quad w_2\\ \vdots \\w_n - \alpha \cfrac{\partial}{w_n} J(\vec{w},b) \quad \overset{\text{sub}}{\rightsquigarrow} \quad w_n\\ b - \alpha \cfrac{\partial}{\partial b} J(\vec{w},b) \quad \overset{\text{sub}}{\rightsquigarrow} \quad b\end{matrix}$$

# 1. 4 Convergence of Gradient Descent
---
In univariate regression, the cost function forms a convex surface that has the only global minimum, whereas in multiple regression, the surface can be more complex, potentially making gradient descent more difficult.
### Learning Curve
By plotting the learning curve, we can diagnose its convergence. The learning curve represents how the cost changes over iterations.

If the learning curve gradually decreases and then stabilizes, we can say that the gradient descent has converged. In this case, you can automatically diagnose convergence by setting a small ε and checking if the cost does not change by more than ε over a certain number of iterations.

The possible problems are...
- Learning curve increases or oscillates : try with lower learning rate
- Learning curve decreases but slowly : try with larger learning rate

### Local Minima
A complex cost function could have a surface with several local minima. That is, even if gradient descent converges, there could be areas with a significantly lower cost. In order to avoid local minima, it is important to experiment with and compare various initial parameter values.

# 1. 5 Feature Engineering
---
Feature engineering, when applied appropriately to the situation, can enhance the performance of a model. There are various types of feature engineering, but here we will focus on two of them. To illustrate, I will present specific problem cases.
## 1. 5. 1 Feature Scaling
---
Suppose that we have 2 features as follow:

$x_1$ : size of house $(300\leq x_1\leq 2000)$ 
$x_2$ : number of bedrooms ($0 \leq x_2 \leq 5$)

Because of the difference of range between the two features, $w_1$ needs to be very small for their contributions to the prediction to be comparable. In this case, we attempt to adjust $x_1$​ by applying various proper factors that can be used in this problem.

1. Maximum Normalization
$$300 \leq x_1 \leq 2000\quad\rightsquigarrow\quad \cfrac{300}{\underset{i\in\mathbb{N}_m}{\max}x^{(i)}_1} \leq \cfrac{x_1}{\underset{i\in\mathbb{N}_m}{\max}x^{(i)}_1} \leq \cfrac{2000}{\underset{i\in\mathbb{N}_m}{\max}x^{(i)}_1}\quad\implies\quad\cfrac{300}{2000}\leq x_1'\leq\cfrac{2000}{2000}$$
2. Mean Normalization
Let $\mu_1$ be the mean value of the set $\left\{x^{(i)}_1\ |\ i\in\mathbb{N}_m\right\}$. Given $\mu_1=600$,
$$
\begin{aligned}
300 \leq x_1 \leq 2000 \quad &\rightsquigarrow \quad \cfrac{300 - \mu_1}{\underset{i \in \mathbb{N}_m}{\max} x^{(i)}_1 - \underset{i \in \mathbb{N}_m}{\min} x^{(i)}_1} \leq \cfrac{x_1 - \mu_1}{\underset{i \in \mathbb{N}_m}{\max} x^{(i)}_1 - \underset{i \in \mathbb{N}_m}{\min} x^{(i)}_1} \leq \cfrac{2000 - \mu_1}{\underset{i \in \mathbb{N}_m}{\max} x^{(i)}_1 - \underset{i \in \mathbb{N}_m}{\min} x^{(i)}_1} \\
&\quad \implies \quad \cfrac{-300}{1700} \leq x_1' \leq \cfrac{1400}{1700}
\end{aligned}
$$
3. z-score Normalization
Let $\sigma_1$ be the standard deviation of the set $\left\{x^{(i)}_1\ |\ i\in\mathbb{N}_m\right\}$. GIven $\sigma_1=450$,
$$300\leq x_1 \leq 2000 \quad\rightsquigarrow\quad \cfrac{300-\mu_1}{\sigma_1}\leq\cfrac{x_1-\mu_1}{\sigma_1}\leq\cfrac{2000-\mu_1}{\sigma_1}\quad\implies\quad\cfrac{-300}{450}\leq x_1'\leq\cfrac{1400}{450}$$
These 3 types of normalization only change the range of the feature values. But generally, normalization refers to techniques that transform not only the scale of feature values but also their overall distribution. For example, normalizing skewed data to approximate a Gaussian distribution is also a type of feature scaling.
## 1. 5. 2 Polynomial Feature
---
Suppose that we have 2 features as follow:

$x_1$ : width of the house
$x_2$ : depth of the house

Having data on the width and depth of a house can provide meaningful information, but intuitively, the area—obtained by multiplying the two—could be even more helpful for carrying out accurate predictions.

Define $x_3$ as $x^{(i)}_3 = x^{(i)}_1*x^{(i)}_2$ and build a model as $f=w_1x_1+w_2x_2+w_3x_3+b$.

Even in cases where polynomial features may not seem intuitively useful, it is advisable to try various polynomial features. This is because adding higher-order terms allows the model to learn more complex relationships between the feature values. In particular, polynomial features are useful when the data exhibits complex nonlinear interactions, as they enable the model to capture patterns that cannot be captured by simple linear model.

You can even try things like $x_1^2$, $x_1^3 \cdot x_2$​, $x_2^4$​, and so on.

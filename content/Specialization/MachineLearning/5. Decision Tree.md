# 5. 2 Split
---
As seen in the tree above, each node has the most optimal feature to split the data. We will measure the purity of each split by evaluating how the data is mixed after split.
## 5. 2. 1 Entropy ; Impurity
---
If we place "face shape" at the root node, the data will be split into two groups of 7 and 3. Among them, only 4 and 1 data are cats, respectively. For example, the left split has following purity:

$$
p=\cfrac{|\{data|x_2=1 \land y=1\}|}{|\{data|x_2=1\}|}=\cfrac{4}{7}
$$

To represent the impurity each set, we define **entropy** function $H:[0,1]\rightarrow[0,1]$ as follow:

$$
H(p)\coloneqq p\log{p}-(1-p)\log(1-p)
$$

Hence, each split of feature "Face Shape" has two $H(4/7)\approx0.99$,  $H(1/3)\approx0.92$, respectively.
## 5. 2. 2 Information Gain
---
To classify cats, we need to construct the tree in a way that reduces entropy. At this point, we combine the entropy of the two splits into a single value to easily compare which feature can reduce more entropy. To give more weights to splits with large number of datas in determining the impurity, we will first use a **weighted average**
$$
\cfrac{|\{data|x_1=1\}|}{|S|}\cdot H(4/7)+\cfrac{|\{data|x_1=0\}|}{|S|}\cdot H(1/3) = \frac{7}{10}H(4/7)+\frac{3}{10}H(1/3)
$$
And then, to measure how much the impurity has reduced after the split, information gain is (entropy of parent node) - (weighted average). In the example, since the parent node is the root node, its entropy is $H(5/10)$. Therefore, if we place the second feature at the root node, the information gain is as follow:

$$
H(5/10)-\left\{\frac{7}{10}H(4/7)+\frac{3}{10}H(1/3)\right\}\approx0.03
$$

The first and third features yield information gains of about 0.28 and 0.12, respectively. Thus, using the first feature -ear shape- at the root node leads to the purest split.
## 5. 2. 3 Stopping Splitting
---
At each split, we will recursively apply the same principle, choosing the feature that yields the highest information gain. Now, take a look at when we should stop splitting the tree.

1. When node is 100% single class
2. When the tree exceeds the preset maximum depth
	- To prevent overfitting
3. When improvement in purity score are less than the threshold
4. When number of examples in an node is less than the threshold

# 5. 3 Feature Problem
---
So far, we have constructed the tree using only binary features. Now, let’s take a look at how to build a tree with the following features.

## 5. 3. 1 Multiple Categorial Feature
---
Suppose the feature "Ear shape" can take on the values pointy, floppy, and oval. While split into multiple branches can be a solution, we'll introduce an alternative method : **one-hot encoding**.

Instead of assigning values from 0 to 2, we will separate this feature into 3 feature as below:

1. Is ear shape pointy?
2. Is ear shape floppy?
3. Is ear shape oval?

Then, pointy, floppy, and oval ones can take (1, 0, 0), (0, 1, 0), and (0, 0, 1), respectively. Through this, we can build a tree with same method that we learned.

## 5. 3. 2 Continuous Valued Feature
---
Suppose that we have a feature for the weight of the animal. To split into 2 groups, we will transform the weight feature into a question like: "Does the weight exceed a specific threshold?".

To determine the best threshold for a continuous feature like weight, we follow these steps:

1. Sort the data based on the weight values.
2. Try every possible threshold between adjacent values, and for each threshold:
    - Split the data into two groups: one with weight ≤ threshold, and one with weight > threshold.
    - Compute the information gain from this split.
3. Select the threshold that gives the highest information gain.

Therefore, acontinuous valued feature can be transformed into a binary feature using a threshold, and then used in the tree.

# 9. 0 Introduction
---
Reinforcement learning is a type of machine learning that focuses on decision making by autonomous agents. Suppose autonomous helicopter control model. So far, we could try such a supervised learning as below:

$$
\begin{matrix}
x(\text{status of HC}) & \rightarrow & y(\text{action of HC}) \\
\text{position} & & \text{h/l altitude} \\
\text{orientation} & & \text{l/r angle} \\
\vdots & & \vdots
\end{matrix}
\rightsquigarrow \text{Not proper}
$$
Instead of this, we could set a **reward function** that give
	reward when auto pilot respond properly, and
	penalty when auto pilot respond improperly or crashed.

With precisely constructed reward function, we would optimize the model way maximizing rewards.

## 9. 0. 1 Mars Rover Problem
---
>Problem Statement : Exploring the mars with autonomous agent and it is in discrete state as below.

![figure9-1][ML9-1.png]

Rover can make only left & right movement at each action step. Where should rover go?

We define **terminal state** at which can't move anymore. (It can be described no fuel situation, trunk limit, etc.) And since every movement has opportunity costs, we define **discount factor** $0<\gamma<1$ which effect the total return(reward) $R$ as follow:

$$
R=R_1+\gamma R_2+\gamma^2 R_3+\cdots\quad\text{(until terminal state)} 
$$

So, moving 4>3>2>1 with $\gamma=0.9$ gets total return $R=0+0\cdot(0.9)+0\cdot(0.9)^2+100\cdot(0.9)^3=72.9$. 

And then, we need to find a **policy** $\pi$ s.t. $\underset{s}{\text{state}}\rightarrow\underset{\pi}{\text{policy}}\rightarrow\underset{a}{\text{action}}$.

For example, if $\gamma=0.5$, then rover maximizes its reward when it move as follow:
	$\pi(2)=(\text{go left})\text{, since }R=50$
	$\pi(3)=(\text{go left})\text{, since }R=25$
	$\pi(4)=(\text{go left})\text{, since }R=25$
	$\pi(5)=(\text{go right})\text{, since }R=20$

So, the goal of reinforcement learning is to find a policy $\pi$ that tells what action $a$ to take in every state so as to maximize the return.

# 9. 1 State Action Value Function
---
State Action Value Function or **Q-function**, $Q(s,a)$ is the return that rover gets when...
- start in state $s$
- take the action $a$, once
- then behave "most optimally".

With $\gamma=0.5$, we can calculate manually about our problem defining $\leftarrow$ and $\rightarrow$ as actions that moves left and right, respectively.

$$
\begin{matrix}
Q(2,\leftarrow)=50 & Q(3,\leftarrow)=25 & Q(4,\leftarrow)=12.5 & Q(5,\leftarrow)=6.25\\
Q(2,\rightarrow)=12.5 & Q(3,\rightarrow)=6.25 & Q(4,\rightarrow)=10 & Q(5,\rightarrow)=20\\
\end{matrix}
$$

The best possible return from state $s$ is $\arg\max_a Q(s,a)$, where $a$ is the best possible action in state $s$. Then we can map $\pi$ from $s$ to $a$ s.t. $\pi(s)=a$.

# 9. 2 Bellman Equation
---
Set $R(s)$ be the reward of state $s$. Then, $Q(s,a)$ is recursively defined as follow:

$$
Q(s,a)=R(s)+\gamma\max_{a'}Q(s',a')
$$

where, $s'$and $a'$ is a state and an action after taking action $a$ in state $s$. This equation is called **Bellman Equation**. There is one more situation to consider in this problem.

# 9. 3 Stochastic Environment
---
Realistically, our rover can make a random mistake because of slipperiness or some problems. Suppose your rover mis-move with 10% chances. Then the optimal move can be changed. What we want to do is to calculate the average of returns.

For example, in process of finding the value of $Q(4,\leftarrow)$,
- optimal move is : $(4,\leftarrow)$ > $(3,\leftarrow)$ > $(2,\leftarrow)$ > $R=\gamma^3 100$
- but these can be happen : $(4,\underset{mis}{\rightarrow})$ > $(5,\rightarrow)$ > $R=\gamma^2 40$
- or : $(4,\leftarrow)$ > $(3,\underset{\text{mis}}{\rightarrow})$ > $(4,\leftarrow)$ > $(3,\leftarrow)$ > $(2,\leftarrow)$ > $R=\gamma^ 5 100$
- or : $(4,\leftarrow)$ > $(3,\leftarrow)$ > $(2,\underset{\text{mis}}{\rightarrow})$ > $(3,\underset{\text{mis}}{\rightarrow})$ > $(4,\leftarrow)$ > $(3,\leftarrow)$ > $(2,\leftarrow)$ > $R=\gamma^ 6 100$
	
	$\vdots$

In these cases, what we want to do is compute all of the possible returns in stochastic environments and calculate average of these as follow:

$$
E\left[R_1+\gamma R_2+\gamma^2R_3+\cdots\right] := P(\text{no miss})Q(4,\leftarrow)+P(\text{1 miss})Q(4,\leftarrow,\text{1 miss})+\cdots
$$

In rare situation such as 100-times mistake can be ignored in computation because the probability and discount factor converges to 0 so, we may ignore cases that is lower than specific probability.

Conclusively, modifying the words, the goal of reinforcement learning is to choose a policy $\pi(s)=a$ that serve an action $a$ to take in state $s$ so as to maximize the **expected return**.

$$
\rightsquigarrow Q(s,a)=R(s)+\gamma \mathbb{E}\left[\max_{a'}Q(s',a')\right]
$$

Now we can map the policy $\pi$ with state 4 to the action $\leftarrow$, because $Q(4,\leftarrow) \approx 75.4$ and $Q(4,\rightarrow)\approx60.5$.

Most of problems are no like to previous problem, even original Mars Rover problem, which has discrete single value of state (1, 2, ..., 6). Suppose 2-D the lunar lander problem and the lander module can have following continuous vectorized state:

$$
\begin{rcases}
\text{position of } & x,y\\
\text{angle of } & \phi\\
\text{velocity of } & \dot{x},\dot{y}\\
\text{angular velocity of } & \dot{\phi}\\
\text{which leg is landed} & l,r
\end{rcases}
\text{ are all continuous, }s\text{ can be express as} :
\left(\begin{smallmatrix}
x\\
y\\
\phi\\
\dot{x}\\
\dot{y}\\
\dot{\phi}\\
l\\
r
\end{smallmatrix}\right)
$$

We can take 4 action, no move, thrust left, main, and right at once so $a$ can be expressed one-hot encoding vector.

## 9. 4. 2 Deep Q-Learning(DQN)
---
What we want to do is to build a model that predicts $\hat{y}=Q(s,a)$ from $s,a$ as follow:

$$
x=\underset{\text{12 units}}{
\begin{pmatrix}
s\\
a
\end{pmatrix}}
\rightarrow\text{DL layer}\rightarrow\cdots\rightarrow\underset{\text{1 unit}}{\bigcirc}\rightarrow \hat{y}=Q(s,a)
$$

First, randomly initialize parameters and $Q_{\text{target}}$ using parameters.

Next, we will make the module to simulate the landing process by itself. The module will choose actions based on target $Q$ function for every state ([[#9. 5. 2 ε-greedy policy|not always when using ε-greedy policy]]) until terminal state. Therefore, actions will be chosen randomly in first episode of simulation.

- When simulation starts, save the transition that contains the chosen action $a_0$ in initial state $s_0$, $R(s_0)$, and following state $s_1$ as a single tuple $(s_0,a_0,R(s_0),s_1)$.

- Repeat this until terminal state with saving transitions from $(s_1,a_1,R(s_1),s_2)$ to $(s_{T-1},a_{T-1},R(s_{T-1}),S_T)$, which are stored in a memory called **Replay Buffer**.

Then, fit parameters with samples of replay buffers of the initial episode and $Q_{\text{target}}$ as follow:

$$
\begin{aligned}
& y(s,a):=
\begin{cases}
R(s) & \text{, if }s'\text{ is terminal state}\\
R(s)+\gamma\mathbb{E}\left[\max_{a'}Q_{\text{target}}(s',a')\right] & \text{, otherwise}
\end{cases}\\
& \text{DQN Loss} = \left(Q_{\text{new}}(s,a)-y(s,a)\right)^2
\end{aligned}
$$

After optimizing $w$ and $b$, set $Q_{\text{new}}$ to be a new target function and repeat.

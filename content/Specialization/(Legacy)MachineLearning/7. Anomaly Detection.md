# 7. 0 Introduction
---
 Anomaly Detection is literally to distinguishing some abnormal data points. Following example can be detected as anomalies:

- A sharp increase in temperature is detected in factory sensor data.
- Thousands times more requests than usual in website log data.
- $10,000 credit card transaction occurred

These anomalies can be reported to API server and should be verified manually. This time we will detect anomalies with **Gaussian Distribution**.

# 7. 1 Density Estimation
---
**Density estimation** assumes that a **clean** training set follows a **Gaussian distribution**, and sets a threshold to detect anomalies.

## 7. 1. 1 Data Preprocessing
---
Density estimation typically requires two steps of data preprocessing, excluding dimensionality reduction:

1. Outlier Removal
If outliers remain in the training set, they can distort the estimated high-likelihood region. Therefore, density estimation is generally recommended to remove as many outliers as possible from the training set before fitting the model.

2. Normalization
Let’s assume that dimensionality reduction has already left us with the most dominant features. Since density estimation relies on the assumption of a **Gaussian distribution**, normalization is required not only in terms of scale of the, but also in terms of **distributional shape**.

![figure7-1](/static/MLAttachedFiles/ML7-1.png)

For instance, if a feature $x_1$ in the training set follows a skewed distribution like the histogram on the left, it can be transformed to approximate a Gaussian-like distribution as shown on the right, by applying the log-transformation ($\log(x_1)$).

Other scaling functions such as $\log(x_1+\alpha),\ \sqrt[\alpha]{x_1}$​, and similar nonlinear transformations may also be applied depending on the data characteristics.

With the data properly preprocessed, let’s take a look at the actual process of density estimation.

## 7. 1. 2  Process of Density Estimation
---
Suppose that for the preprocessed training set $\{\vec{x}^{(1)},\cdots,\vec{x}^{(m)}\}$, each sample $\vec{x}^{(i)}$ has $n$ features and the steps are as follow:

1.  Estimate the distribution parameters : **mean** ($\mu$), **variance** ($\sigma^2$)
$$
\vec{\mu}=\cfrac{1}{m}\sum^m_{i=1}\vec{x}^{(i)}\qquad\vec{\sigma}^2=\cfrac{1}{m}\sum^m_{i=1}\left(\vec{x}^{(i)}-\vec{\mu}\right)^2
$$
2. Compute the Probability Density
Assuming the features are independent, the overall probability is calculated as the product of individual probabilities. Then for test data $x$,

$$
p(x)=\prod^n_{j=1}p(x_j,\mu_j,\sigma^2_j)=\prod^n_{j=1}\cfrac{1}{\sqrt{2\pi}\sigma_j}\exp\left(\cfrac{-(x_j-\mu_j)^2}{2\sigma^2_j}\right)
$$
3. Set the threshold.
Set the threshold $\epsilon$ based on domain knowledge or validation set so report $x$ as an anomaly if $p(x)<\epsilon$. In this case, the validation set should contain a certain amount of anomalies so that metrics (Precision & Recall, $\text{F}_1$ score, ...)  can be used to adjust the threshold. It also serves to better reflect the actual situation.

# 7. 2 Comparison between Anomaly Detection and Supervised Learning
---
Let's compare when each algorithm is more useful.
1. Anomaly Detection
	- No need of target values
	- Very small number of positive samples
	- Various types of anomalies -> Hard to learn from positive samples what anomaly looks like

2. Supervised Learning
	- Need target values
	- Large number of positive and negative samples
	- Enough positive samples to get a sense of what positive sample looks like
